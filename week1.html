What is Interval DP — the one idea
Interval DP solves problems where the answer for a range [i, j] depends on answers for sub-ranges inside it. The key decision is always: "what happens last in the range [i,j]?" Fix that last operation, split the range, and combine sub-answers.
Why "last operation"? Because the last operation has no dependencies — everything else is already solved. Working backwards from the last operation makes the recurrence clean.
The 4 questions to define any interval DP
Q1
What does dp[i][j] mean?
State definition first. Always. "dp[i][j] = minimum cost to solve the problem on range [i,j]." If you can't complete this sentence precisely — don't code yet.
Q2
What is the last operation on [i,j]?
For matrix chain: last multiplication. For burst balloons: last balloon popped. For merge stones: last merge. This is the "k" you iterate over to split the range.
Q3
How do sub-ranges combine?
dp[i][j] = min/max over all k of (dp[i][k] + dp[k+1][j] + cost(i,j,k)). The cost function is problem-specific — what does it cost to perform the last operation given the sub-results?
Q4
What order to fill the table?
Always fill by increasing length. Length 1 first, then length 2, up to length n. Because dp[i][j] always depends on strictly shorter sub-ranges.
The 3 classic interval DP shapes — know all 3
1
Split at k — matrix chain / merge stones
dp[i][j] = min over k in [i, j-1] of dp[i][k] + dp[k+1][j] + cost. You split the range at every possible k and pick the best split. Cost depends on the full range [i,j].
for len = 2 to n:
  for i = 0 to n-len:
    j = i + len - 1
    for k = i to j-1:
      dp[i][j] = min(dp[i][j], dp[i][k] + dp[k+1][j] + cost(i,j))
2
Last element at k — burst balloons
dp[i][j] = max over k in [i,j] of dp[i][k-1] + dp[k+1][j] + cost(k, i, j). Here k is the LAST element removed — so [i,k-1] and [k+1,j] are already empty when k is removed. This is the "reverse thinking" insight.
for len = 1 to n:
  for i = 0 to n-len:
    j = i + len - 1
    for k = i to j:
      // k is the LAST balloon popped in [i,j]
      dp[i][j] = max(dp[i][j],
        dp[i][k-1] + dp[k+1][j] + val[k]*val[i-1]*val[j+1])
3
Character matching — strange printer / palindrome
dp[i][j] built from dp[i+1][j], dp[i][j-1], and dp[i+1][j-1] based on whether s[i]==s[j]. The "last operation" is deciding how the boundary characters relate. Used in strange printer, palindrome partitioning, minimum insertions.
for len = 1 to n:
  for i = 0 to n-len:
    j = i + len - 1
    if s[i] == s[j]:
      dp[i][j] = dp[i+1][j-1] // boundary chars match
    else:
      dp[i][j] = 1 + min(dp[i+1][j], dp[i][j-1])
Universal interval DP template — memorise this
// Step 1: initialise dp table
vector<vector<int>> dp(n, vector<int>(n, 0));

// Step 2: base case — single elements (length 1)
for (int i = 0; i < n; i++)
  dp[i][i] = base_case_value;

// Step 3: fill by increasing length
for (int len = 2; len <= n; len++) {
  for (int i = 0; i <= n - len; i++) {
    int j = i + len - 1;
    dp[i][j] = INT_MAX; // or INT_MIN for max problems

    // Step 4: try every split point k
    for (int k = i; k < j; k++) {
      dp[i][j] = min(dp[i][j],
        dp[i][k] + dp[k+1][j] + cost(i, j, k));
    }
  }
}

// Answer is always dp[0][n-1]
return dp[0][n-1];
What changes per problem — only these 3 things
1
Base case value
Single element cost. Matrix chain: 0 (one matrix needs no multiplication). Merge stones: 0 (one pile needs no merge). Strange printer: 1 (one char needs one print).
2
Cost function
What does it cost to perform the last operation on [i,j] after sub-ranges are solved? Matrix chain: p[i]*p[k+1]*p[j+1]. Merge stones: sum(i,j) — prefix sum. Burst balloons: val[k]*val[i-1]*val[j+1].
3
Split range for k
Split: k from i to j-1 (left and right must both be non-empty). Last element: k from i to j (k is the last one, so sub-ranges can be empty). Character match: no k, just check s[i]==s[j].
Common bugs in interval DP
B1
Filling by i,j instead of by length
If you fill row by row — dp[i][j] might use dp[i][k] where k > i, which isn't computed yet. Always fill by increasing length.
B2
Wrong k range
For split problems: k goes i to j-1 (both halves non-empty). For last-element problems: k goes i to j. Off by one here changes the answer completely.
B3
Not initialising dp[i][j] before inner k loop
Set dp[i][j] = INT_MAX (or INT_MIN) before the k loop. Otherwise you're taking min/max with garbage values.
B4
Integer overflow in cost
When cost involves multiplication (matrix chain, burst balloons) — values can overflow int. Use long long or cast explicitly.
