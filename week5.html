State machine DP — the stock problem family
Core idea
Define states explicitly, then write transitions
Each day you're in one of a fixed number of states. Define what each state means. Then write: "to be in state X on day i, I must have been in state Y or Z on day i-1." That's your recurrence.
States for stock problems
Held / sold / rest — varies by problem
// With cooldown:
held[i] = max(held[i-1], rest[i-1] - price[i])
sold[i] = held[i-1] + price[i]
rest[i] = max(rest[i-1], sold[i-1])
answer = max(sold[n-1], rest[n-1])

Bitmask DP — when n ≤ 20
Core idea
Represent subset of visited/used elements as a bitmask
dp[mask][i] = optimal answer when we've processed the elements in mask and last processed element is i. Iterate over all 2ⁿ masks. For each mask, try adding each unvisited element.
// Template
dp[1<<i][i] = base_case for each starting i
for mask = 0 to (1<<n)-1:
  for i = 0 to n-1:
    if mask has bit i set:
      for j = 0 to n-1:
        if mask does NOT have bit j:
          dp[mask|(1<<j)][j] = optimal(dp[mask][i] + cost(i,j))
Signal
n ≤ 20 + "assign each element to exactly one group/task"
Whenever you see n ≤ 20 and the problem involves optimal assignment or ordering — bitmask DP. The 2ⁿ states represent which elements have been handled so far.

Sweep line — extending what you know about intervals
Core idea
Convert intervals to events, sort, process linearly
Every interval generates two events — start and end. Sort all events by time. Sweep through: on start event, something increases. On end event, something decreases. The answer is often the maximum value seen during the sweep.
// Meeting Rooms II template
vector<pair<int,int>> events;
for each interval [s,e]:
  events.push_back({s, 1}); // start
  events.push_back({e, -1}); // end
sort(events);
int rooms = 0, max_rooms = 0;
for auto [time, type] : events:
  rooms += type;
  max_rooms = max(max_rooms, rooms);
