The core idea of DP on trees
Tree structure = recursive structure. Every tree problem is naturally solved bottom-up — children first, then parent. DP on trees just means: what information do I need from each subtree to answer the parent's question?
The return value of your DFS function IS your DP state. Design the return value carefully — it must contain everything the parent needs to compute its own answer.
The 4 questions for any tree DP problem
Q1
What does the DFS return?
This is your DP state. "Return the height of this subtree." "Return {max_robbed_with_root, max_robbed_without_root}." "Return {camera_state, min_cameras}." Design this before writing any code.
Q2
What is the base case?
Null node — what do you return? Single leaf — what do you return? These two base cases cover 90% of tree problems. Get them right before the recursive case.
Q3
How does parent combine children's results?
Given left subtree returns X and right subtree returns Y — what does current node compute and return? This is your recurrence. Write it in one line before coding.
Q4
Is the answer at the root or across all nodes?
Some problems: answer = what root returns. Other problems (diameter, max path sum, binary cameras): answer = global variable updated at every node. Decide which before coding — this changes your function structure completely.
The 3 tree DP shapes — know all 3
1
Return single value — height, sum, count
DFS returns one number. Parent combines with simple math. Examples: max depth, diameter (global ans updated), path sum exists.
int dfs(TreeNode* node) {
  if (!node) return 0;
  int left = dfs(node->left);
  int right = dfs(node->right);
  ans = max(ans, left + right); // global update
  return 1 + max(left, right); // return to parent
}
2
Return multiple values — state machine on tree
DFS returns a pair or struct — multiple states per node. Examples: House Robber III (rob/not rob), Binary Cameras (covered/not covered/has camera). Parent picks best combination of children's states.
// House Robber III
pair<int,int> dfs(TreeNode* node) {
  // returns {rob_root, skip_root}
  if (!node) return {0, 0};
  auto [lr, ls] = dfs(node->left);
  auto [rr, rs] = dfs(node->right);
  int rob = node->val + ls + rs;
  int skip = max(lr,ls) + max(rr,rs);
  return {rob, skip};
}
3
Rerooting — answer depends on full tree not just subtree
Two-pass DFS. Pass 1: compute subtree values bottom-up. Pass 2: propagate parent info top-down to get full-tree answer at each node. Examples: Sum of Distances in Tree, Count Nodes Equal to Average of Subtree.

Trie — 3 operations to know cold
struct TrieNode {
  TrieNode* children[26] = {};
  bool isEnd = false;
};

void insert(string& w) {
  TrieNode* cur = root;
  for (char c : w) {
    if (!cur->children[c-'a'])
      cur->children[c-'a'] = new TrieNode();
    cur = cur->children[c-'a'];
  }
  cur->isEnd = true;
}

bool search(string& w) {
  TrieNode* cur = root;
  for (char c : w) {
    if (!cur->children[c-'a']) return false;
    cur = cur->children[c-'a'];
  }
  return cur->isEnd;
}

bool startsWith(string& p) {
  TrieNode* cur = root;
  for (char c : p) {
    if (!cur->children[c-'a']) return false;
    cur = cur->children[c-'a'];
  }
  return true;
}

Binary Camera — 3 states (hardest tree DP of week)
State 0
Node is NOT covered, no camera
Node needs to be covered by parent. Return cost to optimally cover subtree given this node is uncovered.
State 1
Node IS covered, no camera here
Node is covered by a child's camera. Return cost to optimally cover subtree.
State 2
Node HAS a camera
Camera here covers this node, parent, and children. Return cost including this camera.
This is the hardest problem of the week. Don't start it until House Robber III and Max Product Split are solid.
