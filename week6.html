The root cause of your confusion
Duplicate handling is not one problem — it's three completely different problems that look similar on the surface. Mixing them up is why you get confused.
Type 1
Generate all UNIQUE subsets/permutations from array with duplicates
Tool: Sort + skip duplicate at same level
Type 2
COUNT distinct subsequences of one string in another
Tool: 2D DP with careful recurrence
Type 3
COUNT distinct subarrays with some property
Tool: Contribution counting or hashset
The one question that separates all three types
Ask
Am I generating or counting? And what's the source of duplicates?
Generating + duplicates in input → Type 1 (backtracking + skip). Counting subsequences → Type 2 (DP on two strings). Counting distinct subarrays → Type 3 (contribution or hashset). These three never mix. Identify which type before doing anything else.
Type 1 — Generating unique results from duplicate input
Root cause
Why duplicates appear in output
When you have `[1,1,2]` and pick the first `1`, then later pick the second `1` at the same recursion level — you get the same subset twice. The two `1`s are identical in value so the results are indistinguishable.
Fix
Sort first. Skip duplicate at same level.
After sorting, identical values are adjacent. At each recursion level, if the current element equals the previous element AND the previous element was NOT used in this call — skip it. This ensures each value is used at most once per position at each level.
// Subsets II — unique subsets from array with duplicates
sort(nums.begin(), nums.end()); // MUST sort first

void backtrack(int start, vector<int>& curr) {
  result.push_back(curr);
  for (int i = start; i < nums.size(); i++) {
    // Skip duplicate at same level
    if (i > start && nums[i] == nums[i-1]) continue;
    curr.push_back(nums[i]);
    backtrack(i+1, curr);
    curr.pop_back();
  }
}
Key line
i > start AND nums[i] == nums[i-1]
Both conditions together. `i > start` means we're not at the first element of this level — so a previous identical element was already tried at this level. `nums[i] == nums[i-1]` confirms it's a duplicate. Together: "this exact value was already tried at this position in the recursion tree — skip."
Permutations with duplicates — different fix
Why subsets fix doesn't work for permutations
In subsets, you move forward (i starts from start). In permutations, you pick any unused element each time. The `i > start` trick doesn't apply.
Fix for permutations
Sort + skip if same value already used at this level
sort(nums.begin(), nums.end());
vector<bool> used(n, false);

void backtrack(vector<int>& curr) {
  if (curr.size() == n) { result.push_back(curr); return; }
  for (int i = 0; i < n; i++) {
    if (used[i]) continue;
    // Skip: same value, previous not used = tried at this level already
    if (i > 0 && nums[i]==nums[i-1] && !used[i-1]) continue;
    used[i] = true;
    curr.push_back(nums[i]);
    backtrack(curr);
    curr.pop_back();
    used[i] = false;
  }
}
Key line
nums[i]==nums[i-1] AND !used[i-1]
`!used[i-1]` means the previous identical element is NOT in the current path — so it was already tried and backtracked at this level. Trying the current one would produce a duplicate permutation. Skip it.
Type 2 — Count distinct subsequences (DP)
Core problem
How many distinct subsequences of T exist in S?
Classic 2D DP. dp[i][j] = number of distinct subsequences of T[0..j-1] in S[0..i-1].
// dp[i][j] = ways to form T[0..j-1] using S[0..i-1]
dp[i][0] = 1; // empty T always matched once
dp[0][j] = 0; // non-empty T can't match empty S

for i = 1 to m: // S length
  for j = 1 to n: // T length
    dp[i][j] = dp[i-1][j]; // don't use S[i]
    if S[i-1] == T[j-1]:
      dp[i][j] += dp[i-1][j-1]; // use S[i] to match T[j]
Why this handles duplicates
The DP naturally counts distinct positional matches
If S = "rabbbit" and T = "rabbit" — the three b's create multiple ways to match, all counted separately. You never need to explicitly deduplicate because dp[i][j] counts ways using position, not value. Each position is unique even if values repeat.
Count distinct subsequences of a single string
Problem
Count all distinct subsequences of string S
Different from above — only one string. dp[i] = number of distinct subsequences using S[0..i-1].
dp[0] = 1; // empty subsequence
last[c] = -1 // last position each char was seen

for i = 1 to n:
  dp[i] = 2 * dp[i-1]; // include or exclude S[i]
  if last[S[i]] != -1:
    dp[i] -= dp[last[S[i]]-1]; // subtract duplicates
  last[S[i]] = i;
Why subtract
When S[i] appeared before at position j — all subsequences ending at j get duplicated
Doubling counts all old subsequences twice — once without S[i] and once with it. But if S[i] appeared before, all subsequences formed before that previous occurrence already had S[i] added — they're now counted twice. Subtract dp[j-1] to remove those duplicates.
Type 3 — Count distinct subarrays
Approach 1 — HashSet
For small constraints — store each subarray's hash
Generate all subarrays, hash each one, insert into set. Answer = set size. O(n³) or O(n² log n) — only works for small n. But conceptually clean.
Approach 2 — Contribution counting
For each right endpoint j, count NEW distinct subarrays ending at j
A subarray ending at j is new if it doesn't match any subarray ending at j that was already counted. The number of NEW subarrays ending at j = j - last[hash of subarray]. This is the sliding window + hashmap approach.
Approach 3 — Suffix array
For string problems — total subarrays minus LCP sum
Total distinct substrings = n*(n+1)/2 - sum of LCP array. Advanced — only needed for very specific problems. Know it exists but don't prioritize.
The at-most-K trick — most common distinct subarray pattern
Pattern
Count subarrays with exactly K distinct = atMost(K) - atMost(K-1)
Counting "exactly K" directly is hard. But "at most K" is a clean sliding window. The subtraction gives exactly K. This trick applies to: exactly K distinct characters, exactly K odd numbers, exactly K zeros.
int atMost(int k) {
  int left = 0, count = 0, res = 0;
  unordered_map<int,int> freq;
  for (int right = 0; right < n; right++) {
    if (++freq[nums[right]] == 1) count++;
    while (count > k) {
      if (--freq[nums[left]] == 0) count--;
      left++;
    }
    res += right - left + 1; // all subarrays ending at right
  }
  return res;
}
// Answer: atMost(k) - atMost(k-1)
