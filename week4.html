Design problems — how to think about them
Design problems test one thing: can you pick the right data structure combination to achieve required time complexity? Every design problem is a list of operations with time constraints. Your job is to find a structure that satisfies all constraints simultaneously.
Before coding any design problem — list every operation and its required complexity. Then ask: what data structure gives O(1) or O(log n) for ALL of them? Sometimes one structure isn't enough — combine two.
The 3 questions for any design problem
Q1
What operations are needed and at what complexity?
List every operation explicitly. "insert O(1), delete O(1), getRandom O(1)." This is your constraint set. Every design decision must satisfy all constraints simultaneously.
Q2
What single structure almost works — and what does it fail on?
HashMap gives O(1) insert/delete but O(n) random. Array gives O(1) random but O(n) delete. Identifying the failure tells you what second structure to add to compensate.
Q3
How do the two structures stay in sync?
When you use two structures — every operation must update both consistently. The bug in almost every design problem is one structure getting out of sync with the other. Write the sync logic explicitly before coding.
Two Heaps — when and why
When
Whenever you need the median dynamically
Or more generally: whenever you need to split a stream of numbers into two halves and query the boundary between them efficiently. Median stream, sliding window median, IPO problem.
Structure
Max-heap for lower half + Min-heap for upper half
Lower half max-heap: top = largest of smaller numbers. Upper half min-heap: top = smallest of larger numbers. Median = top of max-heap (odd total) or average of both tops (even total).
// Invariant to maintain always:
// 1. max_heap.size() >= min_heap.size()
// 2. max_heap.size() - min_heap.size() <= 1
// 3. max_heap.top() <= min_heap.top()

void addNum(int num) {
  max_heap.push(num);
  min_heap.push(max_heap.top()); max_heap.pop();
  if (min_heap.size() > max_heap.size())
    { max_heap.push(min_heap.top()); min_heap.pop(); }
}
double findMedian() {
  if (max_heap.size() > min_heap.size())
    return max_heap.top();
  return (max_heap.top() + min_heap.top()) / 2.0;
}
LRU Cache — the most asked design problem
Operations
get O(1), put O(1)
Need O(1) lookup → HashMap. Need O(1) insertion and deletion in order → Doubly Linked List. HashMap stores key → node pointer. List maintains LRU order.
Key insight
Dummy head and tail nodes eliminate edge cases
Never check if list is empty. head.next = most recent. tail.prev = least recent. Every insert goes after head. Eviction removes tail.prev.
// Structure
unordered_map<int, Node*> cache;
Node *head, *tail; // dummy nodes

// get: find in map, move to front, return val
// put: if exists move to front, else insert at front
// if over capacity, remove tail.prev
LFU Cache — harder than LRU
Structure
Three HashMaps + min_freq tracker
key_val: key → value
key_freq: key → frequency
freq_keys: frequency → ordered set of keys (insertion order)
min_freq: current minimum frequency
Key insight
When frequency increases, move key to new frequency bucket
On access: remove key from freq_keys[old_freq], add to freq_keys[old_freq+1]. If freq_keys[min_freq] is now empty — increment min_freq. On eviction: remove first key from freq_keys[min_freq].
Max Frequency Stack — elegant design
Structure
freq map + group map + max_freq tracker
freq: element → current frequency
group: frequency → stack of elements at that frequency
max_freq: current maximum frequency
Key insight
Push element to ALL frequency groups up to its current freq
When you push x with freq=3, x appears in group[1], group[2], group[3]. Pop from group[max_freq] — the top is always the most frequent, most recent. Decrement max_freq if that group becomes empty.
LRU Cache — full implementation
struct Node {
  int key, val;
  Node *prev, *next;
  Node(int k, int v): key(k), val(v), prev(nullptr), next(nullptr) {}
};

class LRUCache {
  unordered_map<int, Node*> cache;
  Node *head, *tail;
  int cap;

  void remove(Node* node) {
    node->prev->next = node->next;
    node->next->prev = node->prev;
  }
  void insertFront(Node* node) {
    node->next = head->next;
    node->prev = head;
    head->next->prev = node;
    head->next = node;
  }
public:
  LRUCache(int capacity): cap(capacity) {
    head = new Node(0,0); tail = new Node(0,0);
    head->next = tail; tail->prev = head;
  }
  int get(int key) {
    if (!cache.count(key)) return -1;
    remove(cache[key]); insertFront(cache[key]);
    return cache[key]->val;
  }
  void put(int key, int value) {
    if (cache.count(key)) remove(cache[key]);
    else if (cache.size() == cap) {
      cache.erase(tail->prev->key);
      remove(tail->prev);
    }
    cache[key] = new Node(key, value);
    insertFront(cache[key]);
  }
};
Insert Delete GetRandom O(1)
class RandomizedSet {
  unordered_map<int,int> val_to_idx;
  vector<int> vals;
public:
  bool insert(int val) {
    if (val_to_idx.count(val)) return false;
    vals.push_back(val);
    val_to_idx[val] = vals.size()-1;
    return true;
  }
  bool remove(int val) {
    if (!val_to_idx.count(val)) return false;
    int idx = val_to_idx[val];
    vals[idx] = vals.back(); // swap with last
    val_to_idx[vals[idx]] = idx; // update moved element
    vals.pop_back();
    val_to_idx.erase(val);
    return true;
  }
  int getRandom() {
    return vals[rand() % vals.size()];
  }
};
